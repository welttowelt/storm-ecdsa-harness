# Lane Playbooks

## PUA/PIP Lane

When a lane repeats itself:

1. Name the failed assumption.
2. State the evidence that falsified it.
3. Produce three materially different hypotheses.
4. Pick the cheapest bounded check.
5. Define the stop condition before running it.

## Research Lane

Research output is useful only when it becomes source-bound:

- public source or paper reference,
- source file/function mapping,
- candidate source hash if applicable,
- q/T or other relevant economics,
- restore, phase, ancilla, and support obligations,
- toy falsifier or cheap local check,
- handoff owner.

## Worker Lane

Worker commands must include:

- source/base,
- exact task,
- range or shard policy,
- validator,
- stop condition,
- output location,
- no-submit acknowledgement.
